# 検証プロトコル

## 1. 目的

このプロジェクトの係数、介入方法、教材評価が実際の能力・得点改善と関係するかを検証します。単発の成功例を一般化しません。

## 2. 検証単位

### 生徒内検証

同じ生徒について、介入前後を比較します。

- 同程度の難易度
- 同じ能力コード
- 制限時間条件の一致
- 未使用問題
- 可能な範囲で同じ時間帯・媒体

### 生徒間検証

十分な件数が集まるまで探索的分析に留めます。生徒間のpriority_score比較は行いません。

## 3. 仮説の証拠レベル

| level | 意味 |
| --- | --- |
| hypothesis | 理論・経験から置いた未検証仮説 |
| observed_once | 1回の観察と一致 |
| replicated | 同一生徒で複数回再現 |
| supported | 複数の未使用測定で予測と一致 |
| rejected | 事前の成功条件を満たさなかった |
| inconclusive | データ不足・条件不一致 |
| superseded | より良い説明へ置換 |

教材の感想だけでsupportedへ変更しません。

## 4. 最小実験

1. 一つの主要能力を選ぶ。
2. 介入前に2回以上測定する。
3. 主指標を一つ決める。
4. 副指標と悪化させてはいけない指標を決める。
5. 介入期間と週間時間を事前登録する。
6. 成功条件を事前登録する。
7. 2から4週間介入する。
8. 未使用類題で2回以上再測定する。
9. 結果、交絡要因、判断を記録する。

## 5. 指標例

| 能力 | 主指標 | 保護する指標 |
| --- | --- | --- |
| LEX | 文中語義の正答率・回答時間 | 1週間後再現率 |
| SYN | 構造特定率 | 意味要素の保持 |
| RDN | 制限時間内正答率 | 時間無制限正答率 |
| QST | 根拠一致率 | 本文理解度 |
| LST | script gap | 初見情報再現率 |
| WRT | 要素充足率 | 文法エラー率 |
| SPK | 応答適合率 | 理解可能性 |
| EXE | 未着手問題数 | 全体正答率 |

速度だけ上がって正答率が下がる場合は成功としません。

## 6. 暫定成功条件

例:

- accuracyが10ポイント以上改善し、2回再現する。
- response_timeが15%以上短縮し、accuracyが悪化しない。
- retentionが80%以上となり、未使用類題へ転用する。
- timed_gapが10ポイント以上縮小し、untimed得点が悪化しない。
- 過去問の同形式で回収点が増える。

これらは一律基準ではありません。開始時の水準、問題数、測定誤差に応じて事前に調整します。

## 7. 交絡要因

- 問題難易度の違い
- 同じ問題の記憶
- 学校・塾で同時に行った学習
- 睡眠・体調・試験時間帯
- 採点者の違い
- 教材への慣れ
- 語数・音声速度・テーマ知識
- 本人の動機や試験緊張

交絡要因が大きい場合はinconclusiveとします。

## 8. priorityモデルの検証

週ごとに次を比較します。

- 予測したhours_to_next_levelと実績
- priority上位能力への配分
- 能力測定値の変化
- 模試・過去問得点への転用
- 優先度を上書きした判断
- 完了率と継続性

係数変更は、一件ごとの結果ではなく複数週の傾向から行います。

## 9. レビュー周期

- 毎週: 実行可能性と短期指標
- 4週ごと: 能力レベルと介入効果
- 模試・過去問ごと: 本番形式への転用
- 学期・入試終了後: モデル係数とケーススタディ

## 10. 結論の表現

避ける表現:

- この方法なら必ず伸びる
- 全受験生に最適
- この教材が最強

推奨表現:

- この条件の生徒では改善と一致した
- 現段階では支持されるが未検証部分がある
- 別の入試形式・学力層では再検証が必要
