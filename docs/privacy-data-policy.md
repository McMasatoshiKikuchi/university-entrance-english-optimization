# 個人情報・データ管理方針

## 1. GitHubへ登録しない情報

- 本名
- 生年月日
- 住所、電話番号、メールアドレス
- 学校名、クラス、出席番号
- 受験番号、出願番号
- 大学・学校サイトのログイン情報
- GitHubトークンやAPIキー
- 顔写真、学生証、受験票
- 面接・発話の録音・録画本体
- 医療・障害・家庭環境の詳細
- 市販教材、模試、過去問の本文・画像・PDF
- 第三者の個人情報

非公開リポジトリであっても登録しません。

## 2. 登録できる情報

- 仮名のstudent_id
- 能力コード
- 得点、正答率、所要時間
- 失点分類
- 教材名と実施範囲
- 大問・設問番号
- 学習時間
- 仮説と介入結果
- 志望校の公開されている入試情報

## 3. 仮名化

- student_idから本人を推測できないようにする。
- 本人との対応表はGitHub外で管理する。
- ケーススタディは複数の特徴を組み合わせ、個人を特定できない形にする。
- 学校名や地域を分析に不要なら記録しない。

## 4. 音声・作文

スピーキング録音、答案画像、作文原稿には個人情報が含まれる可能性があります。

- GitHubには評価値と能力コードだけを登録する。
- 本体を保存する場合は、本人・保護者の同意、保存場所、保存期間、閲覧者を決める。
- 不要になった本体は削除する。
- 公開共有しない。

## 5. 保存期間

- 日次の細かいログ: 意思決定に不要になったら集約後に削除可能
- 週次・月次集計: 検証に必要な期間だけ保持
- ケーススタディ: 十分に匿名化したものだけ保持
- 認証情報: 保存禁止

## 6. 公開範囲

リポジトリを公開へ変更する場合は、全履歴を含めて個人情報・著作物を再点検します。現在ファイルから削除してもGit履歴に残る場合があるため、最初から登録しないことを原則とします。

## 7. インシデント

個人情報や認証情報を誤って登録した場合:

1. 共有・作業を停止する。
2. 認証情報なら直ちに失効・再発行する。
3. GitHub上の表示だけでなく履歴への残存を確認する。
4. 必要な履歴削除を行う。
5. 原因と再発防止策を記録する。ただし漏えい情報そのものは記載しない。
