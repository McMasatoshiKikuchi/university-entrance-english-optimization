# GitHub Pages・表示トラブル対処

## 1. Markdown記号が生データで表示される

症状:

- `# 見出し`や`| 表 |`がそのまま表示される
- 箇条書きが通常の文章として表示される

確認:

1. URLが`.md`で終わっていないか確認します。
2. Pagesでは対応する`.html`ページを開きます。
3. Markdownファイル先頭にPages用設定があるか確認します。

```yaml
---
layout: default
title: ページ名
---
```

## 2. 404 File not found

主な原因:

- リンク先のファイル名や大文字・小文字が違う
- Pagesに存在しない`.md` URLを直接開いている
- `.github/`内の運営用ファイルへPagesからリンクしている
- 更新直後でPagesの再生成が完了していない

対処:

- 一般資料はPagesの`.html`ページを確認します。
- GitHub IssueはPages内ファイルではなく、GitHubのIssue作成URLへリンクします。
- 更新後は数分待って再読み込みします。

## 3. Mermaidがコードのまま表示される

確認:

- コードブロックの言語名が`mermaid`になっている
- 開始と終了のバッククォートが3個ずつある
- Mermaid構文に誤りがない
- `assets/js/mermaid-init.js`が読み込まれている
- 外部スクリプトを遮断するブラウザ設定になっていない

GitHubでは表示され、Pagesだけ表示されない場合は、Pagesの共通レイアウトとMermaid処理を確認します。

## 4. リンクを押すとGitHubログイン画面になる

Issue作成、ファイル編集、修正提案などはGitHubアカウントが必要です。一般閲覧者向けリンクには「GitHubログイン必須」と明記します。

Pagesの閲覧自体にはGitHubアカウントは必要ありません。

## 5. 更新がPagesへ反映されない

1. GitHub上で変更が登録されているか確認します。
2. 数分待ちます。
3. ブラウザを再読み込みします。
4. 古い表示が残る場合は、強制再読み込みまたは別ブラウザで確認します。
5. ファイル先頭のPages用設定と`_config.yml`を確認します。

複数回更新を重ねる前に、現在の公開処理が完了しているか確認してください。

## 6. 表が画面からはみ出す

Pagesでは表を横スクロールできる設計です。スマートフォンでは表の上を左右に動かして確認します。

列が多すぎる場合は、列名を短くする、説明を仕様書へ分ける、用途ごとに表を分割する方法を検討します。

## 7. CSVが文字化けする

- UTF-8対応の表計算ソフトで開く
- CSVの読み込み機能から文字コードUTF-8を指定する
- 保存時もUTF-8を選ぶ

文字化けした状態で上書きしないでください。

## 8. 学習実験Issueが使えない

- GitHubへログインしているか確認します。
- リポジトリのIssues機能が利用できるか確認します。
- 一般閲覧者にはIssue登録を要求しません。
- GitHubアカウントがない場合は、テンプレート内容を手元のノートや文書で使用します。

## 9. 解決しない場合に記録する情報

- 問題が起きたページのURL
- 発生日時
- 使用端末とブラウザ
- 表示されたメッセージ
- 更新したファイル名
- GitHubでは正常か、Pagesだけの問題か
- 個人情報を含まない画面画像

アクセストークン、パスワード、認証コードは記録・共有しないでください。
