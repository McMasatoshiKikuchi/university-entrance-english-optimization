# 学習優先度モデル v0.1

## 1. 目的

診断結果と志望校の入試形式から、次の1週間にどの能力へ何分配分するかを決めます。

このモデルは合格確率を直接予測するものではありません。限られた学習時間の配分を一貫した基準で比較し、毎週修正するための意思決定補助です。係数は未検証の暫定値です。

## 2. 入力

各能力について次を登録します。

| 項目 | 範囲 | 意味 |
| --- | ---: | --- |
| current_level | 0から4 | 診断された現在レベル |
| target_level | 0から4 | 入試形式から決めた必要レベル |
| exam_importance | 0から1 | 志望校群での重要度 |
| transfer | 0.5から1.0 | 他大学・他形式へ転用できる程度 |
| trainability | 0.25から1.0 | 現期間内に改善できる見込み |
| urgency | 0.75から1.5 | 本番までの期限 |
| evidence | 0.5から1.0 | 診断・配点情報の確かさ |
| hours_to_next_level | 1以上 | 次のレベルへ必要と見積もる時間 |
| dependency_bonus | 0から0.25 | 下流能力を解放する価値 |

## 3. 複数志望校の重要度

志望校・方式ごとに進学優先度を0から1で登録し、合計を1に正規化します。

```text
exam_importance_i
= Σ(
    志望校優先度_s
    × 英語の総合点寄与_s
    × 能力重み_s,i / 5
  )
```

- 志望校優先度: 第一志望だけを1にせず、現実的な受験予定全体で配分する。
- 英語の総合点寄与: 英語配点 ÷ 合否判定総点。出願資格のみの場合は別フラグで扱う。
- 能力重み: Phase 3の0から5。大学別プロファイルがあればそちらを優先する。

配点不明の場合は、evidenceを下げて過度な最適化を防ぎます。

## 4. 計算

### 4.1 能力差

```text
gap = max(0, target_level - current_level) / 4
```

現在レベルが目標以上の場合、改善課題ではなく維持課題へ移します。

### 4.2 直接価値

```text
direct_value
= gap × (0.7 × exam_importance + 0.3 × transfer)
```

入試への直接寄与を重視しつつ、複数方式へ転用できる基礎能力も残します。

### 4.3 依存関係ボーナス

```text
dependency_bonus
= min(
    0.25,
    0.25 × 下流能力の最大gap
         × 下流能力のexam_importance
         × dependency_strength
  )
```

例:

- LEX不足がRDN・LST・WRT・SPKを同時に妨げている。
- SYN不足がRDNと記述問題を妨げている。
- RDN不足がQSTや要約を妨げている。

上位問題を解き続けるより前提能力を補う方が合理的な場合に使用します。

### 4.4 生の優先度

```text
priority_raw
= 100
× (direct_value + dependency_bonus)
× trainability
× urgency
× evidence
÷ max(hours_to_next_level, 1)
```

### 4.5 相対スコア

```text
priority_score
= priority_raw / 全能力中の最大priority_raw × 100
```

相対スコアは同一生徒の同一週でのみ比較します。他の生徒との比較には使いません。

## 5. 暫定パラメータ

### urgency

| 本番まで | 値 |
| --- | ---: |
| 181日以上 | 0.75 |
| 91から180日 | 1.00 |
| 31から90日 | 1.25 |
| 30日以下 | 1.50 |

期限が近いだけで、短期改善できない課題を最優先にしないようtrainabilityと併用します。

### evidence

| 根拠 | 値 |
| --- | ---: |
| 公式配点＋複数回の診断・過去問 | 1.00 |
| 公式配点＋1回の診断 | 0.85 |
| 公式情報または診断の片方のみ | 0.70 |
| 自己申告・一般論のみ | 0.50 |

### trainability

| 状態 | 値 |
| --- | ---: |
| 方法が明確で短期改善を確認済み | 1.00 |
| 標準的な介入方法がある | 0.75 |
| 複数原因があり追加診断が必要 | 0.50 |
| 現期間では改善困難 | 0.25 |

## 6. 時間配分

### 6.1 同時実行数

- 週10時間未満: 重点能力は最大2つ
- 週10時間以上: 重点能力は最大3つ
- METAの週次レビューは別枠で確保する

課題を増やすことより、再テストまで完了させることを優先します。

### 6.2 基本配分

1. 週学習時間の10%を復習・再テスト・計画修正へ確保する。
2. 残りを上位能力へ50:30:20で配分する。
3. 重点能力が2つなら60:40とする。
4. 本番形式演習の時間を残り期間別ロードマップに従って確保する。
5. 本番形式演習で出た誤りは翌週のpriority入力へ戻す。

### 6.3 最低配分ルール

- LEXがLevel 2未満でRDN・LST・WRT・SPKの目標が3以上: LEXへ最低20%
- SYNがLevel 2未満でRDN・WRTの目標が3以上: SYNへ最低20%
- 共通テストまたは外部試験でLSTが必要かつLevel 2未満: LSTへ最低15%
- 英作文・面接が課されない方式: WRT・SPKの直接対策は維持または停止を選べる
- 30日以下: 新規の長期教材より過去問・形式演習・再発防止を優先する

最低配分の合計が大きすぎる場合は、重点能力数と目標設定を見直します。

## 7. 優先度を上書きする条件

数式より次を優先します。

- 出願資格となる外部試験スコアの締切が近い。
- 前提能力が不足し、上位演習が成立しない。
- 学校・塾の必須課題があり、入試学習と統合できる。
- 体調不良や学習時間減少により、計画量を維持できない。
- 公式発表により入試科目・配点・方式が変更された。
- 同じ介入を2週間行っても測定値が改善しない。

上書き理由は必ず週次記録へ残します。

## 8. 出力

毎週、次を出力します。

- 優先能力1から3
- 各能力の週間時間
- 具体的な学習行動
- 使用教材・範囲
- 成功条件
- 再テスト日
- 今週行わないこと
- 数式を上書きした判断と理由

## 9. 検証課題

- 係数0.7 / 0.3の妥当性
- hours_to_next_levelの見積精度
- priority_scoreと実際の得点変化の相関
- 生徒が実行可能な重点能力数
- 最低配分ルールの効果
- 診断誤差に対するスコアの感度

十分な運用データが得られるまで、スコアは意思決定補助として扱います。
