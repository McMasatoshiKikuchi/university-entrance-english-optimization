# 初回診断テスト仕様 v0.1

## 1. 目的

初回診断は、生徒を順位付けする試験ではありません。大学受験英語の得点を制約している能力を特定し、次の4週間で優先する学習領域を決めるために実施します。

出力は総合点だけに集約せず、LEX・GRM・SYN・RDN・QST・LST・WRT・EXE・METAの能力プロフィールとして残します。

## 2. 設計原則

- 正答率だけでなく、所要時間、確信度、再現率を測る。
- 知識問題と文中での運用問題を組み合わせる。
- 制限時間あり・なしの差を測る。
- 読解力と設問処理力を分離する。
- 初見結果と24から72時間後の再テストを分ける。
- 志望校で不要な技能は、優先度計算から外せるようにする。
- 著作権のある市販教材・模試・過去問本文は収録しない。
- 問題ごとに測定対象の能力コードを付与する。

## 3. 実施構成

疲労による歪みを減らすため、原則として2回に分けます。

### Session A: 基礎・読解診断 65から75分

| 順序 | モジュール | 時間目安 | 主な能力 |
| ---: | --- | ---: | --- |
| 1 | 事前情報・確信度説明 | 5分 | META |
| 2 | 語彙・熟語 | 10分 | LEX |
| 3 | 文法・語法 | 10分 | GRM |
| 4 | 英文解釈 | 15分 | SYN |
| 5 | 長文読解・設問処理 | 25から35分 | RDN / QST / EXE |

### Session B: 音声・記述・差分診断 45から60分

| 順序 | モジュール | 時間目安 | 主な能力 |
| ---: | --- | ---: | --- |
| 1 | リスニング | 15分 | LST / QST |
| 2 | 英作文・記述 | 20分 | WRT |
| 3 | 読解の時間無制限再分析 | 10から15分 | SYN / RDN / QST |
| 4 | 学習管理質問 | 5から10分 | META |

### Follow-up: 24から72時間後 10から15分

- 語彙・文法の一部を再テストする。
- 英文解釈の類題を実施する。
- 初回の誤答理由を本人が再説明する。
- 単純な答えの暗記を避けるため、可能な限り同一能力の類題を使う。

## 4. 能力別の問題仕様

### LEX: 語彙・熟語

- 単語から中心語義を選ぶ。
- 文中で多義語の意味を選ぶ。
- 派生語の品詞と意味を判断する。
- コロケーション・熟語を完成する。
- 正答までの時間を記録する。

単語単独の知識と文中運用を別スコアにします。

### GRM: 文法・語法

- 文法形式を選択する。
- 誤りを訂正する。
- 文法形式が表す意味を説明する。
- 長文中の文法要素を特定する。

単独文法問題だけで習熟を判定しません。

### SYN: 英文解釈

- 主語、述語動詞、節境界を示す。
- 修飾先と指示語の参照先を特定する。
- 省略・挿入・倒置を含む文の構造を説明する。
- 意味要素の抜けがない和訳または日本語説明を作る。

正誤に加えて、構造、意味保持、表現の3観点で採点します。

### RDN / QST: 長文読解・設問処理

同程度の文章を、制限時間条件と時間無制限条件で比較します。

- 段落ごとの役割を選ぶ。
- 全体要旨を説明する。
- 根拠箇所を示す。
- 内容一致・推論・言い換えを処理する。
- 本文を理解した後、誤答選択肢が誤りである理由を説明する。

時間無制限で改善する場合は速度・試験実行力、改善しない場合は理解・設問処理を優先候補にします。

### LST: リスニング

- 文字なしで語・短文を認識する。
- 会話・説明から条件や数字を抽出する。
- 話者の意図を判断する。
- 音声と図表・選択肢を統合する。
- スクリプト確認後の理解度と比較する。

スクリプト確認後に大幅に改善する場合は音声認識を主要候補とします。

### WRT: 英作文・記述

- 基本文を生成する。
- 日本語を簡略化して英語にする。
- 条件付きの短い自由英作文を書く。
- 短い英文を要約する。
- 自己点検後に修正箇所を記録する。

文法、意味伝達、構成、条件遵守、自己修正の観点を分けます。

### EXE / META

試験中の行動記録と質問票で測ります。

- 大問別の予定時間と実績
- 解答順序
- 保留・捨て問判断
- 見直し方法
- 確信度と正誤の一致
- 普段の復習間隔
- 再テスト実施状況
- 教材の完了・中断理由

## 5. 共通測定値

| 指標 | 定義 |
| --- | --- |
| accuracy | 獲得点 ÷ 満点 |
| response_time | 問題・モジュールの実所要時間 |
| time_ratio | 基準時間 ÷ 実所要時間。上限1.0 |
| confidence | 解答時の確信度。0・25・50・75・100% |
| calibration | 確信度と実際の正誤の一致度 |
| transfer | 単独知識を文中・類題で使えた割合 |
| retention | 24から72時間後に再現できた割合 |
| timed_gap | 時間無制限得点率 − 制限時間得点率 |

確信度は解答後ではなく、採点前に記録します。

## 6. 暫定評価尺度

この尺度は運用データを集めるまでの暫定値です。大学の合格判定には直接使いません。

| Level | 暫定状態 | 目安 |
| ---: | --- | --- |
| 4 | 安定 | 制限時間内で概ね85%以上、類題でも再現 |
| 3 | 実用 | 概ね70から84%、一部に速度・再現の問題 |
| 2 | 不安定 | 概ね50から69%、条件が変わると崩れる |
| 1 | 未成熟 | 概ね30から49%、基礎要素に複数の欠落 |
| 0 | 未形成 | 30%未満または測定不能 |

最終レベルは正答率だけで決めず、速度・転用・再現を含めて判断します。

## 7. 診断の出力

必須出力:

1. 9領域のレベルと根拠
2. 上位3つのボトルネック
3. 維持すべき強み
4. 制限時間による得点低下
5. 確信度の過大・過小評価
6. 次の4週間で行う学習課題
7. 再診断日と成功条件

## 8. 優先順位への変換

診断後、各能力について次を評価します。

```text
優先度
= 得点回収余地
× 志望校での重要度
× 他形式への転用性
× 改善可能性
× 緊急度
÷ 必要学習時間
```

ただし、下位能力が上位能力の前提になっている場合は、単純な得点効率より依存関係を優先します。例えば、語彙認識が成立していない状態で長文速度演習だけを増やしません。

## 9. 適応ルーティング

- LEXとGRMが低い: SYN・RDNの難問を減らし、基礎の追加診断へ進む。
- SYNが低い: 長文得点だけでRDNを判定せず、構文類題を追加する。
- RDNが高くQSTが低い: 選択肢分析と根拠特定を追加する。
- untimed得点が高くtimed得点が低い: RDN-09とEXEを追加測定する。
- 読解が高くLSTが低い: 音声認識と意味保持を分けて追加測定する。
- WRTのみ低い: 文生成、構成、条件遵守を分けて追加測定する。

## 10. 実施条件

- 通知を切り、辞書・参考書を使わない。
- 端末・紙など実施媒体を記録する。
- 中断時間を記録する。
- 採点前に所要時間と確信度を保存する。
- 体調不良や極端な睡眠不足の場合は延期する。
- 問題の出典・利用条件を記録する。
- 同じ問題の短期間での再利用を避ける。

## 11. 現段階の限界

- レベル境界は未検証の暫定値である。
- 問題難易度の等化は未実施である。
- 学年・志望校別の標準値は未作成である。
- 実際の診断問題本文はまだ作成していない。
- 妥当性と信頼性は運用データを用いて検証する必要がある。
