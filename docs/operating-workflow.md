# 運用ワークフロー

## 1. 全体ループ

```text
志望校プロファイル
  ↓
初回診断・模試・過去問
  ↓
能力別優先度
  ↓
週間計画
  ↓
日々の学習セッション
  ↓
類題・模試・過去問で再測定
  ↓
仮説の支持・棄却
  ↓
次週の配分を更新
```

## 2. ID

個人情報の代わりに次のIDを使います。

- student_id: 生徒の仮名ID
- session_id: 1回の学習
- assessment_id: 診断・模試・過去問・類題テスト
- error_id: 一つの失点
- hypothesis_id: 改善仮説
- experiment_id: 仮説を検証する介入
- material_id: 教材名と役割を結ぶID

ID例: `S001`, `SES-20260711-01`, `A-20260715-01`。本名、学校名、受験番号はIDに含めません。

## 3. 最初の設定

1. student_idを決める。
2. 志望校・学部・方式をプロファイルへ登録する。
3. 英語の週間学習可能時間を登録する。
4. 初回診断または直近の客観的結果を登録する。
5. 能力別current_levelとtarget_levelを置く。
6. priority_scoreを計算する。
7. 重点能力を2から3個に絞る。
8. 週間最適化テンプレートを作る。

## 4. 日次

学習終了後に最低限次を記録します。

- ability_code
- intervention
- planned_minutes / actual_minutes
- 完了した問題数・範囲
- 正答数・正答率
- 主な失点分類
- 次の行動
- 再テスト予定日

日記のような長文は必須にしません。意思決定に使わない項目を増やしません。

## 5. 評価時

模試・過去問・類題テストでは次を登録します。

1. assessment_idを発行する。
2. 問題形式、試験プロファイル、得点、時間を記録する。
3. 失点ごとにerror_idと能力コードを付ける。
4. 高確信の誤答を優先して分析する。
5. 問題本文ではなく、大問・設問番号と失点理由だけを記録する。
6. 修正行動と再テスト日を決める。

## 6. 仮説と実験

例:

```text
仮説:
RDN得点が低い主因は読解速度ではなく、
SYN-03の修飾関係の誤認である。

予測:
SYN介入後、未使用長文の一文理解率が上がり、
時間無制限得点と制限時間得点の両方が改善する。
```

仮説を登録してから介入を始め、都合のよい結果だけを採用しないようにします。

## 7. 週次

1. 予定と実績の時間差を確認する。
2. 再テスト未実施を確認する。
3. 能力別accuracy・time・retentionを集計する。
4. 模試・過去問への転用を確認する。
5. 仮説をsupported・rejected・inconclusiveへ更新する。
6. 重点能力を維持・変更・終了する。
7. 次週の週間計画を作る。

## 8. 月次

- 志望校プロファイルと残り日数を更新する。
- 診断レベルを再評価する。
- priority係数の見積りと実績を比較する。
- 教材の継続・停止を判断する。
- 学習時間増加ではなく、得点・速度・再現率の改善を確認する。
- 個人情報や不要な記録を削除する。

## 9. 状態

共通のstatus値:

- draft: 未確定
- active: 実施中
- waiting_retest: 再テスト待ち
- supported: 仮説を支持
- rejected: 仮説を棄却
- inconclusive: 判断不能
- completed: 完了
- paused: 一時停止
- retired: 使用終了

## 10. データ品質

- 日付は`YYYY-MM-DD`。
- 時間は分または秒の列名を明示する。
- 割合は0から1で保存する。
- 未測定は空欄とし、0点と区別する。
- 推測値にはevidence_statusを付ける。
- 同じ能力コードと失点分類を使う。
- 後から値を変更した場合は、理由を週次記録へ残す。
