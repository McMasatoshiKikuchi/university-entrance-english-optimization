# Roadmap

## Phase 1: 構造定義

- [x] プロジェクトの目的と対象を定義する
- [x] 最適化対象を定義する
- [x] 英語能力を10領域へ分解する
- [x] 下位能力コードを定義する
- [x] 初期診断ルールを作る

## Phase 2: 診断設計

- [x] 各能力を測る問題形式を定義する
- [x] 制限時間あり・なしの測定方法を定義する
- [x] 正答率、速度、確信度、再現率の尺度を作る
- [x] 初回診断の所要時間と実施順を決める
- [x] 診断結果の出力形式を作る
- [ ] 著作権上問題のない実問題を作成する
- [ ] 問題難易度を等化する
- [ ] 暫定レベル境界を実データで校正する
- [ ] 再検査信頼性と予測妥当性を確認する

設計仕様は完了していますが、実問題と評価尺度は未検証です。現段階のLevel 0から4は合否判定に使いません。

## Phase 3: 入試方式との接続

- [x] 共通テスト英語の要求能力を分類する
- [x] 国公立二次の代表的な形式を分類する
- [x] 私立一般の代表的な形式を分類する
- [x] 推薦・外部試験利用との接続を整理する
- [x] スピーキング能力を能力マップと診断仕様へ追加する
- [x] 大学・学部・方式別プロファイルのデータ構造を作る
- [ ] 個別大学の配点・形式を公式情報から登録する
- [ ] 過去問分析により暫定能力重みを校正する

方式分類と登録用データ構造は完了しています。個別大学の実データは、対象校を決めた後に年度・学部・選抜区分単位で登録します。

## Phase 4: 学習最適化

- [x] 学習優先度モデルを数値化する
- [x] 能力別の学習方法を整理する
- [x] 教材選定基準を定義する
- [x] 残り期間別ロードマップを作る
- [x] 週次の再計画ルールを作る
- [x] 能力依存関係をデータ化する
- [x] 優先度・教材評価・週間計画のデータ形式を作る
- [ ] 優先度係数を実データで校正する
- [ ] hours_to_next_levelの見積方法を検証する
- [ ] priority_scoreと得点変化の関係を検証する

意思決定モデルと運用形式は完成しています。数値係数は未検証であり、他の生徒との比較や合格確率の推定には使いません。

## Phase 5: 運用と検証

- [x] 学習ログ用CSVを作る
- [x] 模試・過去問分析テンプレートを作る
- [x] 合成の生徒タイプ別ケーススタディを作る
- [x] 仮説・経験則・検証済み知見の状態を定義する
- [x] 仮説・介入実験・週次指標のデータ形式を作る
- [x] 個人情報・著作物の管理方針を作る
- [x] 運用ワークフローと検証プロトコルを作る
- [ ] 実生徒データで診断と得点変化の関係を検証する
- [ ] 優先度モデルを前向きに検証する
- [ ] 改善しなかった事例を含めてケーススタディを更新する
- [ ] 複数の生徒・入試方式で外的妥当性を確認する

運用基盤は完成しています。効果検証はまだ開始しておらず、合成ケーススタディを実証結果として扱いません。
